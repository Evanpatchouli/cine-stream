# 修复方案：播放页续播恢复被历史加载时机吞掉

## 问题

播放页把续播恢复逻辑放在 `video.onLoadedMetadata` 中执行。如果视频元数据先返回、观看历史后返回，那么恢复逻辑只会跑一次空数据分支，最终仍从头开始播放。

## 根因

- 续播恢复依赖两个异步条件：视频元数据和个人观看历史。
- 旧实现只绑定元数据回调，没有在历史晚到时补偿执行。

## 方案

- 把续播恢复从 `onLoadedMetadata` 中拆出，改为独立 `useEffect`。
- `useEffect` 同时依赖：
  - 当前剧集 id
  - 当前剧集历史记录
  - 历史是否已加载完成
  - 视频时长是否可用
- 元数据回调仅负责初始化 `duration/currentTime/volume/muted/playbackRate`，不再承担业务恢复逻辑。
- 续播位置统一由 `position_seconds + duration_seconds` 推导；接近片尾则返回 0，避免反复续播到结尾。

## 适用场景

- 个人观看记录异步加载晚于视频元数据。
- 播放页既要支持首次进入默认剧集，也要支持按历史自动回到上次观看剧集。
