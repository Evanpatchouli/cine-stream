# 修复方案：真机原生 HLS 播放时总时长显示为 00:00

## 问题

桌面端播放正常，但真机浏览器中视频总时长显示为 `00:00`，同时播放器仍可继续播放。

## 根因

- 真机场景更容易走浏览器原生 HLS 分支，而不是 `hls.js` 分支。
- 原生 HLS 在 `loadedmetadata` 时刻不一定已经给出稳定的有限时长，可能暂时返回 `0`、`NaN` 或 `Infinity`。
- 当前播放页主要依赖 `video.duration` 驱动总时长、进度百分比和观看进度上报；一旦浏览器时长无效，UI 和业务逻辑都会一起退化。

## 方案

- 把后端返回的 `episode.duration_seconds` 作为播放器时长兜底来源。
- 前端继续监听媒体元素的真实 `duration`，当浏览器后续给出有限正数时，用真实值覆盖兜底值。
- 所有依赖时长的逻辑统一走“真实时长优先，否则静态时长兜底”：
  - 总时长展示
  - 进度条最大值
  - 续播恢复
  - 观看进度百分比计算
  - 观看历史上报

## 适用场景

- Android / iOS 真机浏览器走原生 HLS，`video.duration` 返回异常值
- HLS 视频本身可播放，但播放器 UI 总时长、进度计算或上报链路异常
